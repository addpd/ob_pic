## ❄️ 一、`新建 文本文档.html`

### 📌 1.1 概述

单文件、零外部依赖（无 npm、无 CDN 脚本）。页面标题为「冰汽时代 Frostpunk - 官方网站」，内容为游戏氛围的宣传式布局：全屏深色背景、Canvas 暴风雪粒子、中央毛玻璃信息卡片与行动按钮。

> 💡 **命名说明**：文件名仍为系统默认的「新建 文本文档」，与内容无关；若用于分享或归档，建议重命名为例如 `frostpunk-landing.html`。

### ✅ 已完成要点

- ✅ **单文件零依赖**：无 npm、无外链脚本与样式，本地双击即可预览。
- ✅ **全屏沉浸布局**：`html/body` 满高、`overflow: hidden`，主内容 Flex 居中。
- ✅ **Canvas 暴风雪**：`Snowflake` 粒子类 + `requestAnimationFrame` 循环，约 200 片雪花，含下落、漂移与边界循环重生。
- ✅ **窗口自适应**：监听 `resize`，同步 `canvas` 尺寸为视口宽高。
- ✅ **冷色氛围底**：画布容器径向渐变（中心 `#1a2a3a` → 边缘 `#050a0f`）。
- ✅ **液态玻璃卡片**：半透明背景、`backdrop-filter` / `-webkit-backdrop-filter` 模糊、描边与内外阴影、`hover` 上移动效。
- ✅ **排版与 CTA**：主副标题、两段中文说明、药丸形「进入新伦敦」按钮及 `hover` 反色高亮。

### 🧱 1.2 技术栈与架构

- **HTML5**：语义简单，`lang="zh-CN"`。
- **CSS**：全部写在 `<style>` 中，无预处理器。
- **JavaScript**：内联 `<script>`，使用 **Canvas 2D API** 与 `requestAnimationFrame` 做动画循环。

### 🏗️ 1.3 页面结构（自上而下）

1. **`<canvas id="blizzard-canvas">`** 🖼️  
   全屏绝对定位，作为雪花绘制层；底层叠加径向渐变（`#1a2a3a` → `#050a0f`），强化寒冷氛围。

2. **`.main-container`**  
   相对定位、`z-index: 2`，在画布之上；Flex 水平垂直居中。

3. **`.glass-card`（液态玻璃卡片）** ✨  
   
   - 半透明深蓝灰背景、`backdrop-filter: blur(16px)`（含 `-webkit-` 前缀）。  
   - 细白描边、内外阴影、圆角 20px。  
   - `hover` 时整体上移 5px，增加轻微动效。

4. **文案区块** 📝  
   
   - 主标题：`冰汽时代`（大写样式、字间距、白色发光文字阴影）。  
   - 副标题：`THE CITY MUST SURVIVE`（游戏经典标语）。  
   - `.description`：两段中文介绍，左对齐，说明世界观与抉择主题。

5. **`.btn` 按钮** 🔘  
   链到 `#` 的「进入新伦敦」；半透明白底、圆角药丸形，`hover` 反色高亮。

### 🎬 1.4 动画与交互逻辑

- **`Snowflake` 类**：每个粒子具有随机位置、半径（约 0.5–3）、垂直速度、水平漂移、不透明度。
- **运动规则**：下落 + 持续向右的微风（`speedX + 1.5`）；超出下边界或左右边界时从对侧/顶部循环重生。
- **数量**：初始化 **200** 个粒子，形成较密的「暴风雪」观感。
- **`resize`**：监听 `window.resize`，将 `canvas.width/height` 设为 `innerWidth/innerHeight`，与 `initSnow()` 配合可在调整后重新分布（代码在 `resize` 后未自动重调 `initSnow`，窗口剧烈改变时粒子分布依赖边界重置逻辑仍可工作）。

### ♿ 1.5 样式与可用性注意点

- `body, html` 使用 `overflow: hidden` 与 `height: 100%`，**页面不滚动**，适合沉浸式首屏。
- 毛玻璃效果依赖浏览器对 `backdrop-filter` 的支持；过旧浏览器可能退化为普通半透明块。
- 字体栈：`'Helvetica Neue', Arial, sans-serif`。

### 👀 1.6 如何预览

双击文件用默认浏览器打开，或将文件拖入 Chrome / Edge 等现代浏览器即可。

## 🛒 二、`ai电商页面.html`

### 📌 2.1 概述

单文件电商风格 **首页原型**：品牌「云端商城」，包含顶栏营销文案、粘性导航、Hero、分类栅格、可筛选商品列表、服务卖点、页脚与订阅表单，以及购物车数量与 Toast 反馈。数据为前端写死的模拟数据，**非真实后端电商**。

### ✅ 已完成要点

- ✅ **技术栈接入**：Tailwind CDN（含 `primary` / `primaryHover` 主题扩展）、Vue 3 全局版 Composition API、Font Awesome 6 图标。
- ✅ **顶栏与导航**：营销通知条；Logo + 桌面端分类链接；大屏圆角搜索框；用户/购物车入口；`sticky` 顶栏；移动端汉堡菜单与展开搜索、导航。
- ✅ **Hero 区**：Unsplash 全宽背景图、暗色遮罩、季节标签、主副标题渐变字、说明文案与双按钮样式。
- ✅ **热门分类**：`categories` 数据驱动，`v-for` + 响应式网格，悬停图片缩放与底部渐变文案层。
- ✅ **商品推荐**：Tab 切换（`activeTab`）；`filteredProducts` 按品类筛选；卡片含图片、新品/特价角标、星级与评价数、现价与划线价；悬停快捷操作层（收藏样式 + 加购）。
- ✅ **购物车反馈**：`addToCart` 累加 `cartCount`、导航栏红色角标；`toggleCart` 有商品时 Toast 摘要；右下角 Toast + `<Transition>` 进出场。
- ✅ **订阅区**：邮箱表单 `@submit.prevent` 触发成功 Toast（前端模拟，无真实接口）。
- ✅ **信任与页脚**：四列服务保障（包邮、正品、退换、客服）；品牌简介、社交占位链接、购物指南/配送链接、订阅表单；底栏版权与支付方式图标展示。
- ✅ **体验细节**：`[v-cloak]` 防闪烁、`scroll-behavior: smooth`、WebKit 自定义滚动条；分类与商品区 `sm` / `md` / `lg` 栅格与导航显隐。

### 🌐 2.2 外部依赖（需联网）

| 资源                                                                          | 用途                                                |
| --------------------------------------------------------------------------- | ------------------------------------------------- |
| `https://cdn.tailwindcss.com`                                               | Tailwind CSS 运行时 JIT（浏览器内生成样式）                    |
| `https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.4.0/css/all.min.css` | 图标（导航、按钮、页脚支付/社交等）                                |
| `https://unpkg.com/vue@3/dist/vue.global.js`                                | Vue 3 全局构建（`createApp` + Composition API `setup`） |
| `https://images.unsplash.com/...`                                           | Hero、分类图、商品图等多张远程图片                               |

⚠️ 离线或 CDN 被拦截时，样式、脚本或图片可能无法加载，页面会残缺或空白。

### ⚙️ 2.3 技术要点

- **Vue 3**：根节点 `#app`，`v-cloak` 隐藏未编译模板；`<Transition>` 包裹右下角 Toast 的进出场动画。
- **Tailwind**：`tailwind.config` 内扩展主题色 `primary`（`#4f46e5`）、`primaryHover`（`#4338ca`）。
- **自定义 CSS**：`[v-cloak]` 隐藏、`scroll-behavior: smooth`、WebKit 滚动条样式。

### 🧩 2.4 页面模块说明

1. **顶部通知条** 📢  
   深色横条：新用户满减活动文案 + 「立即领取」链接（`#`）。

2. **`<header>` 导航** 🧭  
   
   - Logo：云图标 + 「云端商城」。  
   - 桌面端：`md:flex` 显示分类链接；`lg:block` 显示圆角搜索框。  
   - 右侧：用户图标、购物车（`@click="toggleCart"`）、`md:hidden` 汉堡菜单切换 `mobileMenuOpen`。  
   - **移动端**：展开区含搜索框与垂直导航链接。

3. **Hero 区** 🎯  
   Unsplash 全宽背景图 + 暗色遮罩；文案含「2026 夏季新品」标签、主标题与渐变高亮副标题、说明段落、「立即选购」「查看活动」按钮（无实际路由）。

4. **热门分类** 🗂️  
   `v-for="category in categories"`：2×2 / 4 列响应式网格；图片悬停放大、底部渐变遮罩与品类名、商品数量文案。

5. **为您推荐** ⭐  
   
   - **Tab**：`全部商品`、`潮流服饰`、`数码科技`、`家居好物`；`activeTab` 控制高亮。  
   - **`filteredProducts` 计算属性**：选「全部商品」返回全部 `products`，否则按 `product.category` 与当前 Tab 字符串精确匹配筛选。  
   - **商品卡片**：图片、可选「新品」绿标或「特价」红标（二者均为左上角，若数据同时存在会重叠，当前模拟数据未同时出现）、星级（`Math.floor(product.rating)` 决定实心星数量）、评价数、现价/划线原价、`addToCart` 与悬浮层的收藏/加购按钮。

6. **优势保障** 🛡️  
   四列：全场包邮、正品保障、无忧退换、24/7 客服（图标 + 标题 + 简述）。

7. **`<footer>`** 📄  
   品牌简介、微信/微博/TikTok 占位链接、购物指南与配送服务链接列表、邮箱订阅表单（`@submit.prevent="subscribe"`）。

8. **底部版权** ©️  
   `© 2026 云端商城`；支付方式图标为展示用。

9. **Toast** 💬  
   固定右下角；`addToCart` 显示「已将 xxx 加入购物车」，约 3 秒消失；购物车图标在 `cartCount > 0` 时点击显示件数摘要（约 2 秒）；订阅成功显示感谢文案。

### 🔢 2.5 数据与交互行为摘要

- **`cartCount`**：每次 `addToCart` 自增；角标显示在购物车图标上。  
- **无购物车侧边栏/结算页**：仅有数量与 Toast，属于轻量演示。  
- **`subscribe`**：未调用真实 API，仅弹出成功 Toast。  
- **「加载更多商品」**：按钮无 `@click` 逻辑，点击无效果。

### 📱 2.6 响应式断点（Tailwind 惯例）

小屏隐藏桌面导航、显示汉堡菜单；大屏显示完整搜索与导航。栅格在 `sm` / `md` / `lg` 断点切换列数。

### 👀 2.7 如何预览

在联网环境下用浏览器直接打开 `ai电商页面.html`。若使用 `file://` 协议，多数环境下仍可加载 CDN；若遇跨域或混合内容策略问题，可改用本地静态服务器（如 `npx serve`）打开同目录。
